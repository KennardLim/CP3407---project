# CP3407---project

### User Storys

1. Title: AI Market Prediction
Priority: 10
Estimated effort: 5 days
Short description:
As a trader, I want the website to generate a market prediction using an LLM, so that I can receive AI-based support before making a trading decision.

2. Title: Final Predicted Price
Priority: 10
Estimated effort: 4 days
Short description:
As a user, I want the system to provide one final predicted price instead of only a trend, so that I can better understand the expected market movement.

3. Title: Confidence Score
Priority: 20
Estimated effort: 3 days
Short description:
As a user, I want the final prediction to include a confidence score, so that I can judge how reliable the prediction may be.

4. Title: News-Based Prediction Factor
Priority: 10
Estimated effort: 5 days
Short description:
As a trader, I want market news to affect the prediction result, so that the prediction reflects current market events.

5. Title: News Sentiment Analysis
Priority: 20
Estimated effort: 4 days
Short description:
As a user, I want the website to analyse news sentiment using FinBERT, so that I can understand whether recent news is positive, negative, or neutral.

6. Title: XGB Model Integration
Priority: 20
Estimated effort: 5 days
Short description:
As a developer, I want to add an XGB model as one prediction factor, so that the system can combine machine learning output with LLM analysis.

7. Title: Almanac Rule Integration
Priority: 30
Estimated effort: 3 days
Short description:
As a trader, I want almanac rules to be considered in the prediction process, so that historical market patterns can support the final prediction.

8. Title: Ensemble Voting System
Priority: 10
Estimated effort: 6 days
Short description:
As a user, I want the system to combine LLM, XGB, news sentiment, and almanac rules through a reasonable voting or blending method, so that the final prediction is based on multiple factors.

9. Title: Dynamic Blending Weight
Priority: 20
Estimated effort: 5 days
Short description:
As a developer, I want the blending system to use dynamic weights, so that models with better past performance can have more influence on the final prediction.

10. Title: Price Visualisation
Priority: 20
Estimated effort: 4 days
Short description:
As a user, I want to view price data with candlestick charts and line charts, so that I can understand recent market movement visually.

11. Title: Market Sentiment Visualisation
Priority: 30
Estimated effort: 3 days
Short description:
As a user, I want to see visualised market sentiment, so that I can quickly understand the overall mood of the market.

12. Title: News Page
Priority: 30
Estimated effort: 3 days
Short description:
As a user, I want a dedicated news page, so that I can review the news used by the prediction system.

13. Title: Automated Backtesting
Priority: 20
Estimated effort: 6 days
Short description:
As a developer or evaluator, I want the system to automatically backtest prediction results, so that I can evaluate the accuracy of the ensemble model, LLM, and XGB model.

14. Title: Prediction Record Storage
Priority: 20
Estimated effort: 4 days
Short description:
As a developer, I want prediction records to be stored and reused, so that future model evaluation and dynamic weighting can be supported.

15. Title: Backtesting Charts
Priority: 30
Estimated effort: 4 days
Short description:
As an evaluator, I want line charts and an agreement matrix for backtesting outcomes, so that I can compare model performance clearly.

16. Title: Fallback Formula
Priority: 20
Estimated effort: 3 days
Short description:
As a user, I want the system to fall back to the old formula when there are not enough records, so that predictions can still be generated without error messages.

17. Title: Model Explanation
Priority: 30
Estimated effort: 3 days
Short description:
As a user, I want the website to explain the ensemble formula, predicted trading date, market opening time, and market sentiment, so that I can understand how the prediction was produced.

18. Title: User Interface Improvement
Priority: 30
Estimated effort: 4 days
Short description:
As a user, I want the website interface to be improved with better visual design, so that the system is easier and more comfortable to use.

19. Title: Dark and Light Mode
Priority: 40
Estimated effort: 2 days
Short description:
As a user, I want to switch between dark mode and light mode, so that I can use the website in my preferred display style.

20. Title: Website Stability
Priority: 50
Estimated effort: 4 days
Short description:
As a user, I want the website and all new functions to run without errors or server memory issues, so that I can use the prediction system reliably.
